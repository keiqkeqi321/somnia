# 03 - 发版流程

## 一键发版（推荐）

### Windows（支持自动推断版本号）

```powershell
cd somnia

# 预览（自动推断版本号）
.\scripts\release.ps1 -Dry

# 正式发版（自动推断版本号）
.\scripts\release.ps1

# 也可手动指定版本号
.\scripts\release.ps1 0.4.0 -Dry
.\scripts\release.ps1 0.4.0
```

### Linux / macOS（当前需要显式传版本号）

```bash
cd somnia

# 预览
bash scripts/release.sh 0.4.0 --dry

# 正式发版
bash scripts/release.sh 0.4.0
```

## 自动版本推断规则（Windows `release.ps1`）

当不传版本号时，脚本会从以下来源提取历史版本并取最大值：

1. `git tag`（`v*`）
2. `VERSION`
3. `CHANGELOG.md` 中的 `## x.y.z`

然后按规则递增：

- `x.y.0` 到 `x.y.8`：`patch + 1`
- `x.y.9`：进位到 `x.(y+1).0`

示例：

- `0.3.8 -> 0.3.9`
- `0.3.9 -> 0.4.0`

## 发版脚本做了什么

```
.\scripts\release.ps1 [version]
│
├─ ① 检查工作区干净（有未提交更改则中止）
│
├─ ② 解析目标版本
│     ├─ 传参则使用传参
│     └─ 未传参则自动推断下一个版本
│
├─ ③ 更新 VERSION
│
├─ ④ 调用 scripts\sync-version.ps1
│     ├─ 同步 open_somnia/__init__.py
│     └─ 同步 npm/package.json
│
├─ ⑤ 根据 git 自动生成 CHANGELOG 当前版本条目
│     └─ 来源：上一个 tag 到当前 HEAD 的 commit subject
│
├─ ⑥ git add + commit + tag
│
└─ ⑦ git push 分支和 tag → 触发 GitHub Actions CI
      │
      ▼
   CI 自动发布（见 04-CI-CD配置）
```

## 发版前置条件

| 条件 | 说明 |
|------|------|
| 工作区干净 | 没有未提交改动 |
| 版本号格式正确 | 手动传参时必须为 `x.y.z` |
| GitHub Secrets 已配置 | `PYPI_API_TOKEN`（必须） |
| 本地 git remote 可达 | 能 push 到 GitHub |

## 手动发版（不走 CI）

如果 CI 不可用，可以手动发布：

```powershell
cd somnia

# 1. 改版本号并同步
Set-Content VERSION "0.4.0" -NoNewline
powershell -File scripts\sync-version.ps1

# 2. 构建
Remove-Item dist -Recurse -Force -ErrorAction SilentlyContinue
python -m build

# 3. 发布到 PyPI
twine upload dist/* -u __token__ -p "你的PyPI-Token"

# 4. 提交 + tag + push
git add VERSION open_somnia/__init__.py npm/package.json CHANGELOG.md
git commit -m "release: v0.4.0"
git tag v0.4.0
git push origin main
git push origin v0.4.0
```

## 发版检查清单

发版前确认：

- [ ] 代码已合并到目标分支
- [ ] 所有测试通过
- [ ] Git 历史（commit subject）可作为本次 changelog 基础
- [ ] 自动推断版本或手动指定版本符合预期
- [ ] GitHub Secrets 中 `PYPI_API_TOKEN` 有效

发版后验证：

- [ ] GitHub Actions CI 运行成功
- [ ] PyPI 页面更新：https://pypi.org/project/somnia/
- [ ] GitHub Release 已创建
- [ ] 本地验证：`pip install --upgrade somnia`
- [ ] 命令可用：`somnia --help`

## 回滚

如果发布的版本有严重问题：

### 方式一：发 hotfix 版本（推荐）

```powershell
.\scripts\release.ps1
```

### 方式二：从 PyPI 撤回版本

> ⚠️ PyPI 不允许删除版本号，只能 yank（标记为不可安装）

```bash
pip install twine
# 去 PyPI 网页端 Manage -> Release -> Yank
```

### 方式三：删除 git tag

```bash
git tag -d v0.4.0
git push origin :v0.4.0
```
