# 03 - 发版流程

## 一键发版（推荐）

### Windows

```powershell
cd OpenAgent

# 预览（不实际执行）
powershell -File scripts\release.ps1 0.4.0 -Dry

# 正式发版
powershell -File scripts\release.ps1 0.4.0
```

### Linux / macOS

```bash
cd OpenAgent

# 预览
bash scripts/release.sh 0.4.0 --dry

# 正式发版
bash scripts/release.sh 0.4.0
```

## 发版脚本做了什么

```
powershell -File scripts\release.ps1 0.4.0
│
├─ ① 检查工作区干净（有未提交更改则中止）
│
├─ ② 更新 VERSION = 0.4.0
│
├─ ③ 调用 sync-version.ps1
│     ├─ 同步 openagent/__init__.py
│     └─ 同步 npm/package.json
│
├─ ④ 更新 CHANGELOG.md（添加 ## 0.4.0 (日期) 条目）
│
├─ ⑤ git add + commit "release: v0.4.0" + tag v0.4.0
│
└─ ⑥ git push → 触发 GitHub Actions CI
      │
      ▼
   CI 自动执行（见 04-CI-CD配置）
```

## 发版前置条件

| 条件 | 说明 |
|------|------|
| 工作区干净 | 没有 uncommitted changes |
| 版本号格式正确 | 符合 semver：`x.y.z` |
| GitHub Secrets 已配置 | `PYPI_API_TOKEN`（必须） |
| 本地 git remote 可达 | 能 push 到 GitHub |

## 手动发版（不走 CI）

如果 CI 不可用，可以手动发布：

```powershell
# 1. 改版本号
cd OpenAgent
Set-Content VERSION "0.4.0" -NoNewline
powershell -File scripts\sync-version.ps1

# 2. 构建
Remove-Item dist -Recurse -Force -ErrorAction SilentlyContinue
python -m build

# 3. 发布到 PyPI
twine upload dist/* -u __token__ -p "你的PyPI-Token"

# 4. 提交 + tag
git add VERSION openagent/__init__.py npm/package.json
git commit -m "release: v0.4.0"
git tag v0.4.0
git push github.com OpenAgent_by_codex
git push github.com v0.4.0
```

## 发版检查清单

发版前确认：

- [ ] 代码已合并到目标分支
- [ ] 所有测试通过
- [ ] CHANGELOG.md 已补充条目内容（release 脚本只创建标题）
- [ ] 版本号递增正确（PATCH / MINOR / MAJOR）
- [ ] GitHub Secrets 中 PYPI_API_TOKEN 有效

发版后验证：

- [ ] GitHub Actions CI 运行成功
- [ ] PyPI 页面更新：https://pypi.org/project/somnia/
- [ ] GitHub Release 已创建
- [ ] 本地验证：`pip install --upgrade somnia`
- [ ] 命令可用：`somnia --help`

## 回滚

如果发布的版本有严重问题：

### 方式一：发 hotfix 版本（推荐）

```powershell
# 快速修复后发布 PATCH 版本
powershell -File scripts\release.ps1 0.4.1
```

### 方式二：从 PyPI 撤回版本

> ⚠️ PyPI 不允许删除版本号，只能 yank（标记为不可安装）

```bash
pip install twine
twine upload --repository pypi dist/somnia-0.4.0.tar.gz  # 已存在
# 去 PyPI 网页端 Manage → Release → Yank
```

### 方式三：删除 git tag

```bash
git tag -d v0.4.0
git push github.com :v0.4.0
```
