# 任务系统

## 概述

Somnia 的任务系统提供**持久化任务管理**，支持跨会话、跨 Agent 的任务创建、追踪、依赖管理和认领。与 Todo（会话级短期清单）不同，任务系统适合中长期工作规划。

---

## 数据模型

```python
{
    "id": 1,                    # 自增 ID
    "subject": "重构 auth 模块",   # 必填：任务主题
    "description": "...",        # 可选：详细描述
    "status": "pending",         # pending | in_progress | completed | deleted
    "owner": "alice",            # 当前负责人
    "preferred_owner": "alice",  # 偏好负责人（可选）
    "blockedBy": [2, 3],         # 阻塞依赖（任务 ID 列表）
    "blocks": [4],               # 被此任务阻塞的任务
    "created_at": 1718000000.0
}
```

---

## 状态流转

```
pending ──→ in_progress ──→ completed
                │
                └──→ deleted
```

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `in_progress` | 进行中 |
| `completed` | 已完成 |
| `deleted` | 已删除（update 返回 null） |

## 工具

| 工具 | 说明 | 必填参数 |
|------|------|---------|
| `task_create` | 创建新任务 | `subject` |
| `task_get` | 获取任务详情 | `task_id` |
| `task_update` | 更新任务状态/依赖 | `task_id` |
| `task_list` | 列出所有任务 | — |
| `claim_task` | 认领任务 | `task_id` |

### task_create

```json
{
    "subject": "优化搜索性能",
    "description": "当前搜索在大仓库中响应较慢",
    "preferred_owner": "explorer"
}
```

### task_update

```json
{
    "task_id": 1,
    "status": "in_progress",
    "add_blocked_by": [2],
    "add_blocks": [3],
    "preferred_owner": "alice"
}
```

### task_list 渲染格式

```
[ ] #1: 重构 auth 模块 @alice (blocked by: [2])
[>] #2: 编写单元测试 @bob
[x] #3: 更新文档
```

## 任务依赖

通过 `blockedBy` 和 `blocks` 字段实现任务间的阻塞关系：

- `blockedBy`: 当前任务被哪些任务阻塞
- `blocks`: 当前任务阻塞哪些任务

示例：
```python
task_update(task_id=2, add_blocks=[3])  # 任务 2 阻塞任务 3
task_update(task_id=3, add_blocked_by=[2])  # 任务 3 被任务 2 阻塞
```

---

## 任务认领

```
task_claim(task_id=1, owner="alice")
→ "Claimed task #1 for alice"
```

- `owner` 参数可选，默认为当前 actor
- 认领后 `task.owner` 设置为指定值

## 与 Agent Team 的关系

Teammate 在空闲轮询时会：

1. **检查自己拥有的开放任务** → 等待任务完成
2. **扫描可认领的任务** → 自动认领并开始执行

```python
# 自动认领逻辑
claimable = task_store.list_claimable_for(name)
if claimable:
    task = claimable[0]
    task_store.claim(task["id"], name)
    # 自动注入任务内容到 teammate 消息
```

## 与 Todo 的区别

| 维度 | Todo | 任务系统 |
|------|------|---------|
| 生命周期 | 会话级 | 持久化（跨会话） |
| 范围 | 当前工作段 | 中长期规划 |
| 负责人 | 无 | 支持 owner / preferred_owner |
| 依赖关系 | 无 | blockedBy / blocks |
| 最大数量 | 20 项 | 无限制 |
| 团队使用 | 不共享 | 跨 Agent 共享 |
