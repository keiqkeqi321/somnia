# 13-上下文回滚（Checkpoint / Rollback）

## 概述

上下文回滚功能允许用户在对话过程中创建**命名检查点**（Checkpoint），并在需要时**回滚**（Rollback）到任意检查点，恢复会话消息、会话状态和文件变更，无需重启会话或压缩上下文。

典型场景：
- Agent 执行了一连串修改后发现方向错误，回滚到之前的"干净"状态重头再来
- 在高风险操作前创建检查点，操作完成后确认无误再继续
- 多次试验不同方案：checkpoint → 尝试方案 A → 回滚 → 尝试方案 B

## 用户命令

### `/checkpoint [tag]`

创建当前会话状态的命名检查点。

```text
❯ /checkpoint before_refactor
[checkpoint] tag=before_refactor messages=12 tracked_files=3
```

- **tag**：可选的人类可读标签。如果省略，自动生成 `checkpoint_1`、`checkpoint_2` …
- 检查点保存内容：
  - 当前完整消息列表
  - 当前 undo_stack（所有文件变更记录的深拷贝）
  - 会话状态（todo_items、token_usage、rounds_without_todo、`read_file_overlap_state` 等）
  - 最后一条用户消息的摘要（用于回滚时的参考显示）

### `/rollback [tag]`

回滚到指定的检查点。

```text
❯ /rollback
Rollback to Checkpoint
Choose a checkpoint to roll back to. This will revert messages and file changes.
> cp1 | 14:32:05 | 5 msgs | 2 files | implement feature X
  cp2 | 14:35:12 | 8 msgs | 4 files | optimize the helper function
  cancel (default)
```

- 无参数时弹出交互式列表，显示所有检查点的 tag、时间、消息数、文件数和**最后用户消息预览**
- 指定 tag 时直接回滚：`/rollback before_refactor`
- 回滚操作会：
  1. 从 undo_stack 逆序恢复所有检查点之后的文件变更
  2. 截断消息列表到检查点时刻
  3. 恢复会话状态
     包括最近一次 `read_file` overlap coverage
  4. 删除检查点之后创建的孤立检查点

### 外部修改检测

如果回滚时有文件被 Agent 之外的程序修改过（用户手动编辑、git pull 等），系统会在回滚前检测并警告：

```text
[warning] 2 file(s) were modified externally since the checkpoint:
  - src/main.py (content_mismatch: agent writes may have been externally reverted)
  - new_module.py (unexpectedly_missing: Agent created this file but it no longer exists)

External Modifications Detected
These files were changed outside the agent. How do you want to proceed?
> Skip these files (revert only agent-verified files + messages)
  Overwrite them (revert everything to checkpoint state)
  Cancel rollback (default)
```

用户可选择：
- **跳过**：只恢复未被外部修改的文件 + 消息，外部修改的文件保持原样
- **覆盖**：强制恢复所有文件到检查点状态（外部修改会丢失）
- **取消**：放弃回滚

### 检查点存储格式

检查点以 JSON 文件存储在 `{transcripts_dir}/{session_id}.checkpoint.{tag}.json`：

```json
{
  "tag": "before_refactor",
  "timestamp": 1718000000.0,
  "message_count": 12,
  "last_user_message": "refactor the auth module to use OAuth2",
  "messages_snapshot": [ ... ],
  "session_state": {
    "token_usage": {"input_tokens": 5000, "output_tokens": 2000},
    "todo_items": [ ... ],
    "rounds_without_todo": 0,
    "read_file_overlap_state": {
      "source_tool_call_ids": ["call-2"],
      "coverage": {"src/auth.py": [[120, 220]]}
    },
    "latest_turn_id": "turn_abc123",
    "last_turn_file_changes": [ ... ]
  },
  "undo_stack": [
    {
      "turn_id": "turn_xyz",
      "files": [
        {
          "path": "src/auth.py",
          "previous_content": "original content...",
          "existed_before": true
        }
      ]
    }
  ]
}
```

### 文件恢复机制

回滚时从当前 `undo_stack` 顶部向检查点位置逆序遍历：

```
checkpoint undo_stack 长度 = 1
当前 undo_stack 长度 = 4

从 index 3 → 2 → 1 逆序恢复：
  entry[3].files → 写回 previous_content 或删除新建文件
  entry[2].files → ...
  entry[1].files → ...
```

每条 undo 记录包含：
- `path`：文件相对路径
- `previous_content`：agent 修改前的文件内容
- `existed_before`：文件在 agent 操作前是否存在

恢复策略：
- `existed_before=true` → 写回 `previous_content`
- `existed_before=false` → 删除该文件

### 外部修改检测算法

`detect_external_modifications` 对检查点之后 agent 涉及的每个文件执行：

1. 收集该文件在所有 undo 条目中的记录
2. 读取磁盘当前内容
3. 判断逻辑：

| 条件 | 检测结果 |
|---|---|
| `existed_before=false` 且磁盘文件不存在 | `unexpectedly_missing` — agent 创建的文件被外部删除 |
| `existed_before=true` 且磁盘文件不存在 | `unexpectedly_missing` — 原有文件被外部删除 |
| `existed_before=false` 且磁盘内容 == pre-agent 状态 | `content_mismatch` — agent 写入被外部还原 |
| `existed_before=true` 且磁盘内容 == first previous_content | `content_mismatch` — agent 写入被外部还原 |
| 其他 | 不报（agent 写入仍在） |

### 路径安全

所有文件操作严格限制在 `workspace_root` 边界内：
- 所有路径通过 `Path.resolve()` 规范化
- 使用 `Path.is_relative_to(workspace_root)` 检查
- 路径逃逸（如 `../../etc/passwd`）的文件记录会被静默跳过

## 使用示例

### 基本流程

```text
❯ 请帮我重构 auth 模块
● 好的，我来重构 auth 模块...

❯ /checkpoint before_auth_refactor
[checkpoint] tag=before_auth_refactor messages=4 tracked_files=0

❯ 继续重构，把所有 JWT 逻辑移到独立的文件
● 完成。修改了 3 个文件...

❯ 不对，这个方向有问题，回滚吧
❯ /rollback
Rollback to Checkpoint
Choose a checkpoint to roll back to.
> before_auth_refactor | 14:32:05 | 4 msgs | 0 files | 请帮我重构 auth 模块
  cancel (default)

[rollback complete] tag=before_auth_refactor messages_restored=4 files_reverted=3 files_skipped=0 undo_entries_removed=1
```

## 注意事项

1. **检查点不跨会话持久化**：检查点与 session 绑定，新会话无法回滚到旧会话的检查点
2. **回滚删除后续检查点**：回滚到 cp1 时，cp1 之后创建的所有检查点会被自动删除
3. **外部修改可能遗漏**：检测算法只能发现"磁盘内容 == pre-agent 状态"这类明显的外部还原；如果外部修改恰好与 agent 写入内容相同，则无法检测
4. **文件恢复依赖 undo_stack**：如果 agent 的操作未被记录到 undo_stack（如 bash 命令直接修改文件），回滚无法恢复这些变更
5. **并发安全**：回滚命令在 inflight work 完成前会被阻止，避免竞态条件
