# 后台任务

## 概述

后台任务（Background Tasks）允许 Agent 在独立线程中异步执行长时间运行的命令，不阻塞主 Agent Loop，并在完成后自动通知。

---

## 架构

```
主 Agent Loop                    后台线程
    │                              │
    ├── background_run(command)    │
    │       │                      │
    │       ├── 创建 Job 记录      │
    │       ├── 启动线程           │
    │       └── 返回 Job ID ──────▶│
    │                              ├── run_command()
    │                              ├── 捕获输出
    │                              └── 更新状态 ────┐
    │                                               │
    ├── check_background(job_id)                   │
    │       │                                      │
    │       └── 查询 Job 状态 ◀────────────────────┘
    │
    ├── JobStore.notify()         # 通知队列
    │
    └── background_manager.drain()  # 消费通知
```

---

## 工具

### background_run

```json
{
    "command": "python -m pytest tests/",
    "timeout": 300
}
```

- `command`: 必填，Shell 命令
- `timeout`: 可选，超时秒数（默认 `runtime.command_timeout_seconds`）

返回：
```
Background task a1b2c3d4 started: python -m pytest tests/
```

### check_background

```json
{
    "task_id": "a1b2c3d4"
}
```

- `task_id`: 可选，指定任务 ID 查询单个
- 不传：列出所有任务

返回单个：
```
[completed] 15 passed, 2 skipped in 4.2s
```

返回全部：
```
a1b2c3d4: [completed] python -m pytest tests/
e5f6g7h8: [running] npm run build
```

## 任务状态

| 状态 | 说明 |
|------|------|
| `running` | 正在执行 |
| `completed` | 执行完成 |
| `error` | 执行出错 |

---

## 执行细节

```python
def _execute(self, job_id: str, command: str, timeout: int) -> None:
    try:
        completed = run_command(
            command,
            shell=True,
            cwd=self.workspace_root,
            timeout=timeout,
        )
        result = completed.combined_output().strip()
        # 截断到 max_output_chars
        job = self.job_store.update(job_id, status="completed", 
                                    result=result[:self.max_output_chars])
    except Exception as exc:
        job = self.job_store.update(job_id, status="error", result=str(exc))
    
    # 推送通知
    notification = {
        "task_id": job_id,
        "status": job["status"],
        "result": str(job.get("result", ""))[:500],
    }
    self.notifications.put(notification)
    self.job_store.notify(notification)
```

---

## 通知机制

- 任务完成后，结果推入 `notifications` 队列
- Agent Loop 可通过 `drain()` 消费通知
- 通知内容包括任务 ID、状态和结果预览（最多 500 字符）

## 使用场景

- 长时间编译/构建
- 测试套件执行
- 依赖安装
- 数据迁移脚本
- 任何不应阻塞主对话的耗时操作
