## 安装相关

### Q: pip install somnia 报错 "No matching distribution found"

**原因**：Python 版本过低（低于 3.11），或 pip 版本过旧。

**解决**：

```bash
# 检查 Python 版本
python --version   # 需要 ≥ 3.11

# 升级 pip
python -m pip install --upgrade pip

# 重新安装
pip install somnia
```

### Q: 安装成功但 somnia 命令找不到

**原因**：Python Scripts 目录不在 PATH 中。

**解决**：

```powershell
# 查看 Python Scripts 路径
python -c "import sysconfig; print(sysconfig.get_paths()['scripts'])"

# Windows: 添加到系统 PATH
# 例如: C:\Users\你的用户名\AppData\Roaming\Python\Python312\Scripts

# 或直接用 python 调用
python -m open_somnia
```

### Q: pip 有 WARNING: Ignoring invalid distribution

**原因**：之前安装/卸载留下了残留 `.dist-info` 目录。

**解决**：

```powershell
Get-ChildItem "D:\APP\Python\Lib\site-packages" -Directory -Filter "~*" | Remove-Item -Recurse -Force
```

### Q: npx somnia 提示 Python not found

**原因**：系统未安装 Python，或 Python 不在 PATH 中。

**解决**：安装 Python 3.11+，确保 `python --version` 能正常输出。

## 命令相关

### Q: somnia 运行报错 ModuleNotFoundError

**原因**：安装不完整或包版本不匹配。

**解决**：

```bash
pip install --force-reinstall somnia
```

### Q: somnia doctor 显示环境异常

**解决**：根据 `somnia doctor` 输出的具体提示排查，常见问题：
- API Key 未配置
- 网络不通
- Python 版本不匹配

## PyPI 发布相关

### Q: twine upload 报 403 Forbidden

**原因**：
1. Token 无效或过期
2. Token 权限不足（scope 不对）
3. 包名被他人占用

**解决**：
1. 重新生成 PyPI API Token
2. 确认 Token scope 包含 `somnia` 项目
3. 换一个不冲突的包名

### Q: twine upload 报 400 File already exists

**原因**：该版本号已在 PyPI 上发布过，不允许覆盖。

**解决**：递增版本号，修改 `VERSION` 文件后重新发布。

### Q: PyPI 页面长时间不更新

**原因**：PyPI 有 CDN 缓存，通常 1-5 分钟刷新。

**解决**：等待几分钟后刷新，或用 `pip index versions somnia` 命令行查询。

### Q: 包名和已有包太相似被拒

**原因**：PyPI 的名称规范化规则会将 `-` 和 `_` 视为等价，太相似的包名会被拒绝。

**解决**：选择一个完全不同的名字。我们之前就是因为 `openagent` 和已有的 `open-agent` 太相似，改用了 `somnia`。

## CI/CD 相关

### Q: 推送 tag 后 GitHub Actions 没触发

**排查步骤**：

1. 确认 tag 格式正确（必须以 `v` 开头：`v0.4.0`）
2. 确认 tag 已推送到 GitHub：
   ```bash
   git ls-remote --tags github.com
   ```
3. 检查 GitHub 仓库 Settings → Actions → General → Workflow permissions
4. 检查默认分支设置，tag 可能需要从默认分支推送

### Q: CI 中 publish-pypi 失败

**排查**：
1. 检查 `PYPI_API_TOKEN` 是否配置且有效
2. 检查 CI 日志中的具体错误信息
3. 确认 VERSION 文件与 tag 版本号一致

### Q: CI 报错 "Unrecognized named-value: secrets"

**原因**：GitHub Actions 的 `if` 条件中不能直接引用 `secrets`。

**解决**：不要在 job 级别的 `if` 中判断 secrets，改用 `continue-on-error`。

### Q: npm publish 步骤失败

**原因**：`NPM_TOKEN` 未配置或无效。

**当前状态**：已设为 `continue-on-error: true`，不会阻塞后续步骤。配置好 Token 后会自动成功。

## 版本管理相关

### Q: 版本号不一致（三处对不上）

**解决**：

```powershell
# 查看当前版本号
Get-Content OpenAgent\VERSION
Select-String -Path OpenAgent\open_somnia\__init__.py -Pattern "__version__"
(Get-Content OpenAgent\npm\package.json -Raw | ConvertFrom-Json).version

# 重新同步
powershell -File OpenAgent\scripts\sync-version.ps1
```

### Q: git tag 已存在

```
fatal: tag 'v0.4.0' already exists
```

**解决**：

```bash
# 删除本地旧 tag
git tag -d v0.4.0

# 删除远程旧 tag
git push github.com :v0.4.0

# 重新打 tag
git tag v0.4.0
```

### Q: 如何撤销已发布的版本

PyPI **不支持删除版本**，但可以 **yank**（标记为不可安装）：

1. 登录 https://pypi.org
2. 进入 somnia 项目 → Manage → Releases
3. 点击对应版本 → Yank

yank 后用户 `pip install somnia` 不会安装该版本，但 `pip install somnia==0.3.2` 仍然可以指定安装。

## Docker 相关

### Q: docker build 失败

**常见原因**：
1. 网络问题（无法下载基础镜像）
2. Dockerfile 路径不对

**解决**：

```bash
# 确保在 OpenAgent 目录下构建
cd OpenAgent
docker build -t somnia .

# 使用代理
docker build --build-arg HTTP_PROXY=http://proxy:port -t somnia .
```

### Q: docker run 后立即退出

**原因**：没有传入子命令且没有交互终端。

**解决**：

```bash
# 加 -it 参数
docker run -it somnia
```
