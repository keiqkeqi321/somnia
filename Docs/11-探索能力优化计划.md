# 11 - 探索能力优化计划

## 当前状态

旧版探索记忆方案已经下线：

- 不再维护 `exploration_cache`
- 不再维护 repo summary
- 不再提供 `/investigation`
- 不再把探索缓存注入系统提示

原因见 [10-探索能力](./10-%E6%8E%A2%E7%B4%A2%E8%83%BD%E5%8A%9B.md)。

---

## 新计划方向

后续探索能力优化，改成三条独立路线，不再把“探索缓存”和“长期记忆”混为一体。

### 1. 即时探索质量优化

目标：

- 让当轮探索更稳
- 减少路径猜测和重复读取
- 提升 `/scan`、`/symbols`、`tree`、`read_file` 的组合效率

候选项：

- 更强的源码根识别
- 更好的符号结果排序
- 更适合大仓库的扫描摘要
- 对常见项目类型增加更准确的入口提示

### 2. 结构化证据层

目标：

- 让 agent 在当前任务中显式整理“事实、假设、待验证项”
- 但这些内容只属于当前推理链，不自动持久化为长期记忆

候选项：

- 将关键证据组织成 turn 内或 task 内结构
- 在需要时显示给用户，而不是默认长期保存
- 与压缩策略解耦，避免互相污染

### 3. 新记忆系统

目标：

- 单独设计一个真正可控、可验证、可淘汰的记忆系统

设计原则：

- 记忆来源必须可追溯
- 记忆项必须允许失效和清理
- 记忆和即时探索分层
- 不能把未经验证的探索输出来当成稳定事实

---

## 暂不做

以下旧方向暂不继续：

- session 级 exploration cache
- repo 级 summary 持久化
- exploration warning 注入系统提示
- `/investigation` 视图
