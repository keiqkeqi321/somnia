# 02 - 版本管理

## 版本规范

采用 **Semantic Versioning（语义化版本）**：`MAJOR.MINOR.PATCH`

```
0.3.2
│ │ │
│ │ └── PATCH: bug 修复（不影响 API）
│ └──── MINOR: 新功能（向后兼容）
└────── MAJOR: 重大变更（不兼容）
```

当前处于 `0.x.x` 阶段，表示快速迭代中，API 可能变动。

## 版本号管理架构

### 单点维护

**只修改 `VERSION` 文件**，其他位置由脚本自动同步。

```
                 VERSION 文件（手动修改）
                 ┌─────┤ 0.3.2 ├─────┐
                 │      └───────┘     │
                 ▼                    ▼
        pyproject.toml         npm/package.json
        (动态读取 VERSION)      (sync 脚本写入)
                 │
                 ▼
        openagent/__init__.py
        (sync 脚本写入)
```

### 版本号同步脚本

```powershell
# Windows
powershell -File scripts\sync-version.ps1

# Linux/macOS
bash scripts/sync-version.sh
```

脚本会自动将 `VERSION` 文件的值写入：
1. `openagent/__init__.py` 中的 `__version__`
2. `npm/package.json` 中的 `version`

> ⚠️ 一般不需要手动运行 sync 脚本，`release` 脚本会自动调用。

### pyproject.toml 配置

```toml
[project]
name = "somnia"
dynamic = ["version"]          # 版本号动态读取

[tool.setuptools.dynamic]
version = {file = "VERSION"}   # 从 VERSION 文件读取
```

## Git Tag 规范

每个发布版本对应一个 git tag：

```
v0.1.0
v0.2.0
v0.3.0
v0.3.1
v0.3.2
```

### Tag 操作

```bash
# 查看 tag
git tag -l

# 查看某个 tag 详情
git show v0.3.2

# 删除本地 tag
git tag -d v0.3.2

# 删除远程 tag
git push github.com :v0.3.2

# 重新推送 tag
git push github.com v0.3.2
```

## CHANGELOG 维护

文件位置：`OpenAgent/CHANGELOG.md`

格式：

```markdown
# Changelog

## 0.3.2 (2026-04-08)

- 新增 xxx 功能
- 修复 xxx 问题

## 0.3.1 (2026-04-08)

- 修复 CI workflow 语法错误
```

> release 脚本会自动添加版本标题行，但条目内容需要手动补充。
